# REVELation Presentation Format — Quick Reference

> **Do not edit this file.** It is overwritten automatically when the app updates.
> Place any custom instructions or overrides in **[CUSTOM_INSTRUCTIONS.md](CUSTOM_INSTRUCTIONS.md)**.

Target audience: LLMs (Codex, Claude) creating, editing, or translating `.md` presentation files.
Assumes standard Markdown knowledge. Only REVELation-specific behavior is documented here.

Full references: [MARKDOWN_REFERENCE.md](MARKDOWN_REFERENCE.md) · [METADATA_REFERENCE.md](METADATA_REFERENCE.md) · [AUTHORING_REFERENCE.md](AUTHORING_REFERENCE.md) · [VARIANTS_REFERENCE.md](VARIANTS_REFERENCE.md)

---

## File Structure

```
[YAML front matter]
---

[slide content]

***

[slide content]

---

[slide content]

:note:

[speaker notes]
```

A presentation is one `.md` file. YAML front matter is required. Slides follow directly after.

---

## Slide Separators

These override standard Markdown HR behavior when they appear alone on a line:

| Line | Meaning |
|------|---------|
| `***` | Horizontal slide break (new column/stack) |
| `---` | Vertical slide break (child of current column) |
| `:note:` | Speaker notes delimiter — everything below is notes, not slide content |

Use `___` if you need a visible horizontal rule *inside* slide content.

Inside fenced code blocks, `***` and `---` are literal and do not split slides.

## YAML Front Matter

Minimal required fields:

```yaml
---
title: My Presentation
theme: revelation_dark.css
newSlideOnHeading: false
---
```

Set `newSlideOnHeading: false` unless you explicitly want H1/H2/H3 to create slide breaks. When omitted, it may auto-split on headings for legacy compatibility.

### Common fields

| Field | Type | Notes |
|-------|------|-------|
| `title` | string | Display name in presentation library |
| `description` | string | Subtitle/summary |
| `author` | string or `{name, email}` | |
| `theme` | string | CSS filename, e.g. `revelation_dark.css` — **required** |
| `thumbnail` | string | Preview image filename |
| `created` | string | Date, e.g. `2026-01-15` |
| `newSlideOnHeading` | boolean | Default true if omitted; set false when using explicit separators |
| `stylesheet` | string | Extra CSS file relative to presentation folder |
| `scrollspeed` | number | Notes-view auto-scroll speed, e.g. `2.1` |
| `config` | object | Reveal.js config values (see below) |
| `media` | object | Named media aliases (see below) |
| `macros` | object | Named reusable macro blocks (see below) |
| `alternatives` | object or `hidden` | Language variant map (see below) |

### `config` block — Reveal.js options

```yaml
config:
  transition: fade        # none | fade | slide | convex | concave | zoom
  controls: false
  slideNumber: c          # false | true | "c" | "c/t" | "h/v" | "h.v"
  hash: true
  autoSlide: 0            # ms; 0 = off
```

Full option list: https://revealjs.com/config/

## Media Aliases

Define once in front matter, reference anywhere with `media:<alias>`:

```yaml
media:
  intro:
    filename: opener.mp4   # required — file inside _media/
    copyright: Shot by Jane  # optional attribution
```

Usage in slide content:

```markdown
![background](media:intro)
:audio:play:media:intro:
```

---

## Macros

Define reusable slide-setup blocks in front matter. Parameters use `$1`, `$2`, etc.

```yaml
macros:
  fogbg: |
    <!-- .slide: data-background-video="$1" data-background-video-loop -->
    {{attrib:$2}}
```

Invoke with `{{macroname:arg1:arg2}}` or `{{macroname}}`.

## REVELation-Specific Syntax in Slide Content

### Fragment reveal

Append `++` to any list item or line to make it a click-reveal fragment:

```markdown
- First point ++
- Second point ++
```

### Fragment animation

Append `++:animationtype` or `==:animationtype` for appearance-enabled fragment animation special effects. `==:` appear automatically, `++:` appear on click.

```markdown
![](important_image.jpg) ==:flipX

---

## Heading appears automatically with letter rotating effect ==:rotate:let:40

- First point appear automatically ==:rise
- Second point appears on click ++:rise
```

`animationtype` can be one of:

**Fade (rise/drop)**
- `rise` — fade in from below
- `riseBig` — fade in from below (large travel)
- `riseLeft` — fade in from bottom-left
- `riseRight` — fade in from bottom-right
- `drop` — fade in from above
- `dropBig` — fade in from above (large travel)
- `dropLeft` — fade in from top-left
- `dropRight` — fade in from top-right
- `fly` — fade in from left
- `flyBig` — fade in from left (large travel)
- `flyRight` — fade in from right
- `flyRightBig` — fade in from right (large travel)
- `fade` — simple fade in

**Bounce**
- `bounce` — bounce in
- `bounceDown` — bounce in from above
- `bounceUp` — bounce in from below
- `bounceLeft` — bounce in from left
- `bounceRight` — bounce in from right

**Slide**
- `slide` — slide in from left
- `slideRight` — slide in from right
- `slideDown` — slide in from above
- `slideUp` — slide in from below

**Zoom**
- `zoom` — zoom in
- `zoomDown` — zoom in from above
- `zoomUp` — zoom in from below
- `zoomLeft` — zoom in from left
- `zoomRight` — zoom in from right

**Back**
- `backDown` — back in from above
- `backUp` — back in from below
- `backLeft` — back in from left
- `backRight` — back in from right

**Rotate**
- `rotate` — rotate in
- `rotateDownLeft` — rotate in from top-left
- `rotateDownRight` — rotate in from top-right
- `rotateUpLeft` — rotate in from bottom-left
- `rotateUpRight` — rotate in from bottom-right

**Flip / Roll**
- `flipX` — flip in on X axis
- `flipY` — flip in on Y axis
- `flipFull` — full flip
- `roll` — roll in
- `jack` — jack in the box

**Light Speed**
- `lightLeft` — light speed in from left
- `lightRight` — light speed in from right

**Shrink / Skid** *(reveal.js-appearance custom)*
- `shrink` — shrink in
- `shrinkBig` — shrink in (large)
- `shrinkBlur` — shrink in with blur
- `skidLeft` — skid in from left
- `skidLeftBig` — skid in from left (large)
- `skidRight` — skid in from right
- `skidRightBig` — skid in from right (large)

**Attention Seekers** *(loop on existing elements)*
- `hop` — bounce
- `headShake` — head shake
- `heartbeat` — heartbeat pulse
- `pulse` — pulse
- `tada` — tada
- `wobble` — wobble
- `jello` — jello
- `rubber` — rubber band
- `shakeX` — shake horizontally
- `shakeY` — shake vertically
- `flash` — flash
- `swing` — swing

animationtype can be modified by appending :word or :letter, as well as a custom time delay in ms, for example:

```markdown
# This heading zooms word by word very quickly ==:zoom:word:50
```

### `_text_` — verse heading / reference

Single underscores produce a styled block element, not italic:

```markdown
_Verse 1_          ← left-aligned heading/label
_John 3:16_        ← right-aligned when at end of paragraph
```

### `__text__` — underline

Double underscores produce underlined text (not bold as in some Markdown dialects).

### Multiple columns

```markdown
||
Left column content
||
Right column content
||
```

### Attribution

```markdown
:ATTRIB:Photo by Jane Smith
```

### AI marker

```markdown
:AI:
```

Marks the slide with an AI-generated indicator.

## Commands (`:command:` form)

Line-based commands used in slide content. Must be on their own line (except `:note:`):

```markdown
:note:                          ← start speaker notes
:ATTRIB:Photo credit here       ← attribution text
:AI:                            ← AI-content marker

:audio:play:file.mp3:           ← play audio
:audio:playloop:file.mp3:       ← loop audio
:audio:play:media:alias:        ← play via media alias
:audio:playloop:media:alias:    ← loop via media alias
:audio:stop:                    ← stop audio

:transition:fade:               ← set this slide's transition
:animate:                       ← enable auto-animate
:animate:restart:               ← restart auto-animate
:autoslide:12000:               ← auto-advance after 12 s

:bgtint:rgba(0,0,0,0.4):       ← background color tint overlay
:clearbg:                       ← suppress inherited background
:nobg:                          ← suppress inherited dark/light mode
:shiftnone:                     ← suppress inherited left/right shift
:nothird:                       ← suppress inherited upper/lower-third

:hide:                          ← hide slide everywhere
:hide:handout:                  ← hide in handout only
:hide:slideshow:                ← hide in slideshow only

:countdown:from:mm:ss:          ← countdown timer
:countdown:from:hh:mm:ss:
:countdown:to:hh:mm:
```

## Macros (`{{...}}` form) — Top Matter

Macro calls in `{{...}}` syntax are placed at the **top of a slide** (top matter convention). Many are **sticky** — they persist to subsequent slides until reset.

```markdown
{{darkbg}}              ← dark background mode (sticky)
{{lighttext}}           ← light text mode (sticky)
{{upperthird}}          ← upper-third layout (sticky)
{{lowerthird}}          ← lower-third layout (sticky)
{{bgtint:rgba(0,0,0,0.35)}}   ← tint overlay (sticky)
{{transition:fade}}     ← transition (sticky)
{{animate}}             ← auto-animate (sticky)
{{autoslide:15000}}     ← auto-advance 15 s (sticky)
{{attrib:Text here}}    ← attribution (sticky)
{{ai}}                  ← AI marker (sticky)
{{}}                    ← RESET all sticky top-matter state
```

**Reset rule:** `{{}}` on its own line clears all persisted macro state for subsequent slides.

Sticky background resets macros and establishes a new sticky baseline:

```markdown
![background:sticky](stage-loop.mp4)
```

## Magic Image Alt-Text

Special keywords in `![keyword](source)` trigger non-standard rendering:

| Alt-text | Effect |
|----------|--------|
| `background` | Set as slide background (image or video) |
| `background:noloop` | Video background, no loop |
| `background:sticky` | Sticky background — persists to following slides |
| `fit` | Scale image to fill slide |
| `caption:Text here` | Image with caption below |
| `youtube` | Embed YouTube video (use full URL) |
| `youtube:fit` | YouTube embed scaled to fill |
| `web` | Embed webpage in iframe |
| `web:scrollY=500` | Embed webpage scrolled to Y offset |

Source can be a filename, a `media:<alias>`, or a URL.

## Reveal.js Comment Directives
(Please prefer short code, fragment ++ or macros to set these whenever possible and avoid using this syntax unless specifically requested.)

Attach attributes to slides or elements using HTML comments:

```markdown
<!-- .slide: data-transition="fade" data-background-color="#111" -->

## Slide title
```

```markdown
- Point one <!-- .element: class="fragment" -->
- Point two <!-- .element: class="fragment" data-fragment-index="2" -->
```

```markdown
<!-- .stack: data-transition="convex" -->
```

Common `.slide:` attributes: `data-transition`, `data-transition-speed` (`default`/`fast`/`slow`), `data-background-image`, `data-background-video`, `data-background-video-loop`, `data-background-color`, `data-auto-animate`, `data-autoslide`.

## Inter-Presentation Links

Links to other `.md` files load them as a new presentation. Paths resolve from the presentation root, not the current file.

```markdown
[Next deck](other.md)
[Jump to section](other.md#anchor-id)
```

Parent-directory traversal (`../`) is blocked.

---

## HTML in Slide Content

Raw HTML is allowed. Sanitization removes: `<script>`, `<object>`, `<embed>`, `<applet>`, `<base>`, `<meta>`, all `on*` event attributes, `srcdoc`, `javascript:` URLs, and CSS `expression(...)` payloads.

SVG is also allowed as it is interpreted by the browser the same way as HTML. 

Avoid using HTML formatting in slides unless specifically requested to do so.

## Language Variants

### Master file — `alternatives` map

```yaml
---
title: Welcome
alternatives:
  presentation.md: en
  presentation_es.md: es
  presentation_fr.md: fr
---
```

### Variant file — mark as hidden

```yaml
---
title: Welcome (Spanish)
alternatives: hidden
---
```

### Hide master from listing while keeping variants

```yaml
alternatives:
  self: hidden
  i18n/es/welcome.md: es
```

### Runtime selection

Append `?lang=es` to the presentation URL. REVELation resolves it to the mapped file.

### Sync constraint

All variants must have identical slide counts, `***`/`---` structure, and `++` fragment counts for peer sync to work correctly.

If editing a presentation that has multiple-language variants, edit all variants to keep slides in sync.

If variant presentations have different slide layouts, whenever possible try to synchronize slides by adding, removing or changing slide breaks so content aligns across all variants.

## Practical Rules

1. `***` and `---` **must be alone on their own line** to function as separators.
2. `:note:` **must be alone on its own line**.
3. Note and slide separators **must** also have blank lines before and after to function properly.
4. `{{}}` resets all sticky top-matter state — include it at the start of a slide that should have a clean slate.
5. `newSlideOnHeading: false` should almost always be set explicitly when using `***`/`---` separators.
6. Media filenames in `media:` aliases resolve to the `_media/` folder inside the presentation directory.
7. `![background:sticky]()` behaves as a macro and resets all prior sticky macros when applied.
